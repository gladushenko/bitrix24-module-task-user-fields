# Пользовательские поля в задачах Б24

Модуль решает проблему неудобного добавления и редактирования пользовательских полей в карточках задач Б24, а также позволяет добавлять поля других типов, например поле типа `список`, которое недоступно в стандартном функционале Б24.

## Возможности

- вывод выбранных пользовательских полей задач в отдельном блоке карточки задачи;
- привязка блока пользовательских полей к выбранным проектам;
- настройка заголовка блока;
- настройка пользовательских подписей полей в карточках задач;
- скрытие стандартного блока пользовательских полей Bitrix24;
- поддержка списков, логических, строковых, числовых, денежных, ссылочных и дат полей;
- поддержка множественных списочных пользовательских полей в фильтре списка задач;
- редактирование значений полей из карточки задачи;
- показ понятных ошибок при проблемах сохранения в карточке задачи.

## Установка

1. Создайте директорию `modules/gladushenko.taskuserfields` в папке `local`:

   ```text
   /local/modules/gladushenko.taskuserfields
   ```
   
2. Скопируйте файлы модуля в директорию `gladushenko.taskuserfields`.

3. В административной панели откройте:

   ```text
   Marketplace -> Установленные решения
   ```

4. Найдите модуль `Пользовательские поля в задачах Б24` и нажмите `Установить`.

При установке модуль также копирует расширенный компонент списка задач в:

```text
/local/components/bitrix/tasks.task.list
```

Этот компонент нужен для корректной работы пользовательских списочных полей в фильтре списка задач.

5. Перейдите в настройки модуля:

   ```text
   ПП задач Б24 -> Настройки
   ```

## Использование

После сохранения настроек откройте полную карточку любой задачи. Модуль добавит в карточку отдельный блок с выбранными пользовательскими полями.
Для редактирования значения наведите курсор на строку поля и нажмите `Редактировать`. После сохранения модуль отправит значение через AJAX-действие контроллера и обновит отображение поля.
Если поле отмечено в настройках как `Только чтение`, оно отображается в карточке задачи без кнопки редактирования.
Для списочных пользовательских полей модуль улучшает работу фильтра списка задач: множественные списки отображаются как мультиселект.

## Удаление

Удаление выполняется на странице `Установленные решения` административной панели.

   ```text
   Marketplace -> Установленные решения
   ```
